🚨 Breaches & Active Threats
Notable

Accenture confirms breach after hacker lists 35GB of source code and cloud secrets for sale

📅 Jul 8, 2026

A threat actor using the handle "888" posted on the cybercrime forum PwnForums claiming to have stolen source code, RSA and SSH keys, Azure personal access tokens, Azure Storage access keys and configuration files from a private Azure DevOps repository tied to an accenture.com production URL. Accenture told Help Net Security it is "aware of this isolated matter" and has "remediated its source," but has not confirmed the 35GB figure, the exfiltration method, or whether customer data was affected.

Why it matters: Cloud secrets — access tokens, storage keys, service credentials — are now the prime target in these thefts, not just files; any org relying on Azure DevOps should audit token scope and rotation policy today.

China-linked UNK_MassTraction exploits Roundcube n-days to spy on university physics and national-security researchers

📅 Jul 8, 2026

Proofpoint Threat Research has tracked a suspected China-aligned cluster since May exploiting Roundcube webmail flaws (CVE-2024-42009 XSS, CVE-2025-49113 deserialization) against physics, engineering and astrophysics departments at a few dozen US and Canadian universities. The attack chain deploys a stealer called IceCube to harvest credentials, cookies and 2FA data, then drops a webshell or the VShell backdoor — a Go-based tool commonly used by Chinese threat actors.

Why it matters: Academic institutions doing national-security-adjacent research are attractive, under-defended targets — and this campaign shows old, patchable n-day bugs are still enough to get in when patching lags.

🛡️ Vulnerabilities & Patches
Critical

CISA adds max-severity Adobe ColdFusion bug and two Joomla flaws to KEV, deadline July 10

📅 Jul 7, 2026

CVE-2026-48282 (CVSS 10.0), a path-traversal flaw in Adobe ColdFusion, was reportedly under active exploitation within two hours of disclosure and lets unauthenticated remote attackers achieve code execution. CISA added it to the KEV catalog alongside CVE-2026-56290 (Joomlack Page Builder, CVSS 10.0, unauthenticated file upload) and CVE-2026-48908 (JoomShaper SP Page Builder), giving federal agencies until July 10 under Binding Operational Directive 26-04.

Why it matters: A CVSS 10.0 ColdFusion bug exploited within hours of disclosure, plus two more max-severity CMS plugin flaws, makes this one of the tightest KEV patch windows of the month — treat all three as already-compromised if unpatched.

Ubiquiti patches seven critical UniFi flaws, including a CVSS 10.0 bug reachable on ~100,000 exposed endpoints

📅 Jul 8, 2026

CVE-2026-50746 (CVSS 10.0) is an improper access-control flaw in UniFi Connect Application that lets any network attacker run OS commands on the host device with no credentials; fixed in version 3.4.20. Ubiquiti simultaneously patched six more critical bugs (CVE-2026-50747, -50748, -54400, -54402, -55115, -55116) across UniFi Talk, Access, Protect, UniFi OS Server and a wide range of routers, gateways, NAS and surveillance devices. Censys counts roughly 100,000 UniFi OS endpoints reachable from the public internet.

Why it matters: Seven critical bugs across five product lines in one disclosure, on internet-facing management infrastructure, is a mass-patch emergency for anyone running UniFi at the edge — prioritize UniFi Connect and any internet-facing UniFi OS instance first.

Update

Signed "Verified" GitHub commits can be cloned into new hashes without breaking the signature

📅 Jul 8, 2026

Researcher Jacob Ginesin's "Git Hash Chain Malleability" paper shows three practical techniques — covering GitHub's supported ECDSA, RSA, EdDSA and S/MIME signature schemes — that let anyone produce a byte-different commit with an identical tree and metadata, a valid signature, and a fresh "Verified" badge under a new commit hash. Ginesin reported the issue to GNU/Git in January and GitHub in March; neither has shipped a fix.

Why it matters: This breaks the assumption that a verified commit hash is a unique, immutable identifier — anything relying on hash-based blocklists, dependency pinning or reproducible-build verification should treat that assumption as unsafe until forges close the gap.

🤖 AI Security
Major

CISA's KEV catalog gets its first-ever AI agent platform: Langflow's auth-bypass flaw is now a federal must-patch

📅 Jul 8, 2026

CVE-2026-55255, an IDOR flaw in Langflow's /api/v1/responses endpoint that lets an authenticated attacker execute any other user's automation flow, was added to CISA's KEV catalog on July 7 with a Friday deadline. Sysdig had already observed a lone operator weaponizing it together with the unauthenticated RCE CVE-2026-33017 in a sustained late-June campaign; the fix ships in Langflow 1.9.2.

Why it matters: This is the first time an AI agent-building platform has landed on the federal must-patch list — a concrete signal that agentic AI infrastructure is now squarely inside the same exploitation economy as any other enterprise software.

Sophos: AI coding agents now routinely trip the same detection rules built to catch human intruders

📅 Jul 7, 2026

Sophos X-Ops analyzed a week of its own endpoint telemetry and found Claude Code, Cursor and OpenAI Codex repeatedly setting off Credential Access and Execution detections — including PowerShell using DPAPI to decrypt browser credentials, and agents pivoting from a blocked certutil.exe to bitsadmin.exe, a classic living-off-the-land evasion pattern, despite pursuing entirely legitimate tasks.

Why it matters: Security teams need explicit policies and tuned baselines for what AI coding agents are allowed to do on endpoints now, or they'll spend the next year either drowning in false positives or missing the real thing.

Update

Varonis' "Rogue Agent" flaw could have let one edit permission hijack every AI chatbot in a Google Cloud project

📅 Jul 8, 2026

Varonis Threat Labs disclosed "Rogue Agent" in Google's Dialogflow CX: because every Code Block-enabled agent in a GCP project shares one managed Cloud Run environment, an attacker with a single dialogflow.playbooks.update permission on one agent could inject malicious code into a shared, writable file and silently hijack every other agent in the project — reading live conversations, exfiltrating data and sending attacker-written messages. Varonis reported the bug in November 2025; Google shipped a full fix in June 2026.

Why it matters: Shared execution environments behind multi-tenant AI agent platforms are an under-scrutinized blast-radius problem — one compromised or malicious agent builder can compromise every other agent in the same project.
